Aula 8: Unidade III - Integrando os Pilares da POO

Contextualização da Aula Prática

Chegamos a um ponto crucial da disciplina. Após estudarmos os pilares da POO de forma isolada — Encapsulamento, Herança, Polimorfismo, Classes Abstratas e Interfaces — esta aula tem como objetivo conectar todos esses conceitos. Através da construção de um projeto prático e guiado, um sistema bancário simplificado, vamos materializar a teoria e entender como esses pilares trabalham juntos para criar um software organizado, coeso e extensível. Esta é a aula onde a "mágica" da POO acontece, transformando conceitos teóricos em código funcional e bem estruturado.

Plano da Aula Prática (2 h/a)

🎯 O Projeto: Sistema Bancário Simplificado

Descrição do Problema

Precisamos modelar um sistema para um banco que trabalha com diferentes tipos de contas. Por enquanto, teremos Conta Corrente e Conta Poupança. Ambas as contas possuem um número, um titular e um saldo. Ambas permitem depositar e sacar dinheiro. No entanto, elas têm regras diferentes:

  • Uma Conta Corrente cobra uma pequena taxa a cada saque.
  • Uma Conta Poupança não cobra taxa de saque, mas não pode ficar com saldo negativo.
  • Além disso, a Conta Corrente é um serviço tributável, ou seja, paga um imposto anual, enquanto a Conta Poupança é isenta.

🛠️ Roteiro da Atividade Prática (Live Coding)

Etapa 1: Planejamento e UML

Antes de codificar, vamos desenhar um diagrama de classes UML simples para visualizar a estrutura do nosso sistema.

Interface: Tributavel { +getValorImposto(): double }

Classe Abstrata: Conta { -numero: int; -titular: String; #saldo: double; +sacar(): boolean; ... }

|--> Classe: ContaCorrente (herda de Conta, implementa Tributavel)

|--> Classe: ContaPoupanca (herda de Conta)

Etapa 2: A Classe Abstrata Conta

Crie a classe Conta. Ela deve conter os atributos e métodos comuns a todas as contas. O método sacar deve ser abstrato.

Etapa 3: A Interface Tributavel

Crie a interface Tributavel. Ela deve definir o "contrato" para qualquer classe que precise calcular impostos, declarando o método getValorImposto.

Etapa 4: As Classes Concretas ContaCorrente e ContaPoupanca

Crie as duas subclasses. Elas devem herdar de Conta e implementar o método abstrato sacar com suas regras específicas. A ContaCorrente também deve implementar a interface Tributavel.

Etapa 5: Testando Tudo na Classe Banco

Crie uma classe Banco com um método main. Nela, instancie objetos de ContaCorrente e ContaPoupanca, realize depósitos e saques, e imprima os saldos para verificar se as regras de negócio estão funcionando corretamente.

Desafios Adicionais (Opcional)

  • Transferência: Adicione um método transferir(double valor, Conta destino) na classe Conta. Ele deve sacar de uma conta e depositar na outra.
  • Gerenciador de Impostos: Crie uma classe GerenciadorDeImposto com um método calculaTotal(Tributavel t) que recebe um objeto tributável e exibe o imposto devido.

💡 Solução Proposta

Classe Abstrata Conta.java 

public abstract class Conta {
    private int numero;
    private String titular;
    protected double saldo;

    public Conta(int numero, String titular) {
        this.numero = numero;
        this.titular = titular;
    }

    public void depositar(double valor) {
        if (valor > 0) {
            this.saldo += valor;
        }
    }

    public abstract boolean sacar(double valor);

    public double getSaldo() {
        return this.saldo;
    }

    public String getTitular() {
        return this.titular;
    }
}
Interface Tributavel.java 

public interface Tributavel {
    double getValorImposto();
}
Classes Concretas: ContaCorrente.java e ContaPoupanca.java 

public class ContaCorrente extends Conta implements Tributavel {
    public ContaCorrente(int numero, String titular) {
        super(numero, titular);
    }

    @Override
    public boolean sacar(double valor) {
        double taxa = 0.50;
        if (super.saldo >= valor + taxa) {
            super.saldo -= (valor + taxa);
            return true;
        }
        return false;
    }

    @Override
    public double getValorImposto() {
        return super.saldo * 0.01;
    }
}

public class ContaPoupanca extends Conta {
    public ContaPoupanca(int numero, String titular) {
        super(numero, titular);
    }

    @Override
    public boolean sacar(double valor) {
        if (super.saldo >= valor) {
            super.saldo -= valor;
            return true;
        }
        return false;
    }
}
Classe de Teste Banco.java 

public class Banco {
    public static void main(String[] args) {
        ContaCorrente cc = new ContaCorrente(101, "João Silva");
        ContaPoupanca cp = new ContaPoupanca(202, "Maria Santos");

        cc.depositar(1000.0);
        cp.depositar(500.0);

        cc.sacar(200.0);
        cp.sacar(100.0);

        System.out.println("Saldo CC: R$ " + cc.getSaldo());
        System.out.println("Saldo CP: R$ " + cp.getSaldo());
        
        Tributavel t = cc;
        System.out.println("Imposto da CC: R$ " + t.getValorImposto());
    }
}

Checklist da Aula Prática

Ao final desta aula, você deve ter aplicado e entendido como:

  • Usar uma classe abstrata para definir um modelo comum.
  • Proteger dados com encapsulamento (private/protected).
  • Especializar classes usando herança (extends).
  • Criar comportamentos específicos com polimorfismo e sobrescrita de métodos.
  • Adicionar "capacidades" a classes com interfaces.
