Aula 3: Unidade I – Introdução à Programação OO

Ementa e Contextualização

Na aula anterior, vimos a estrutura básica de um programa. Agora, vamos mergulhar no coração da Programação Orientada a Objetos. Os objetos são a base de tudo o que construiremos. Entender como modelar entidades do mundo real (um produto, um cliente, uma transação) em objetos com estado (atributos) e comportamento (métodos) é a habilidade fundamental que nos permite criar software modular, reutilizável e fácil de manter.

🧱 Os Blocos de Construção da POO

1. Classes, Objetos e Instâncias

Segundo Deitel & Deitel (Java: Como Programar), "uma classe descreve os atributos e comportamentos de um conjunto de objetos semelhantes; cada objeto é uma instância dessa classe."

A relação entre classe e objeto é como a de um molde e o item criado a partir dele. A classe define a estrutura e comportamento; o objeto é a concretização que usamos no código.

  • Classe: O projeto, a planta baixa. Define atributos e métodos que todos os objetos daquele tipo terão.
  • Objeto (ou Instância): Uma cópia única criada a partir da classe. Cada objeto tem seu próprio estado.
Carro
- cor : String
- modelo : String
- ano : int
+ ligar() : void
+ acelerar() : void
+ getVelocidadeAtual() : int 

// Defining a simple class Car
public class Car {
    String color;
    String model;
}

// Creating objects (instances) in the main method
public class TestCar {
    public static void main(String[] args) {
        Car car1 = new Car();
        car1.color = "Red";
        car1.model = "Sedan";

        Car car2 = new Car();
        car2.color = "Black";
        car2.model = "SUV";

        System.out.println(car1.color); // Prints "Red"
    }
}

2. Atributos: O Estado de um Objeto

Atributos são variáveis internas da classe que guardam dados. Eles representam o estado de cada objeto.

Exemplo: A classe Carro pode ter os atributos cor, modelo e ano. Cada objeto terá seus próprios valores. 


public class Carro {
    String cor;
    String modelo;
    int ano;
}

Carro meuCarro = new Carro();
meuCarro.cor = "Vermelho"; // State of the object

💡 Dica:

Atributos são geralmente privados em boas práticas de POO e acessados via métodos (getters/setters) para respeitar o encapsulamento.

3. Métodos: O Comportamento de um Objeto

Métodos são funções dentro da classe que definem o comportamento do objeto. Eles podem ler/modificar atributos e interagir com outros objetos.

Na classe Carro, métodos como ligar() ou acelerar() alteram o estado do objeto. 


public class Carro {
    String modelo;
    int velocidade;

    public void acelerar() {
        velocidade += 10;
        System.out.println(modelo + " acelerou. Velocidade: " + velocidade);
    }
}

Carro carro1 = new Carro();
carro1.modelo = "Sedan";
carro1.acelerar(); // Output: Sedan acelerou. Velocidade: 10

📌 Exercício Rápido:

Adicione um método frear() que reduza a velocidade em 5 e teste a chamada em dois objetos diferentes.

💻 Exemplos Práticos: Modelando um Produto

Exemplo 1: Criando a Classe Produto

Vamos modelar um produto de uma loja virtual. A classe terá atributos para nome, preço e um método para exibir suas informações. 


/**
 * Represents a product in an e-commerce system.
 */
public class Produto {
    // Attributes that define the state of a product.
    String nome;
    double preco;
    int quantidadeEmEstoque;

    /**
     * Displays the product's data sheet.
     * This method accesses the object's own attributes.
     */
    public void exibirFichaTecnica() {
        System.out.println("---- FICHA TÉCNICA ----");
        System.out.println("Produto: " + nome);
        System.out.println("Preço: R$ " + preco);
        System.out.println("Estoque: " + quantidadeEmEstoque + " unidades");
        System.out.println("-----------------------");
    }
}

Exemplo 2: Criando e Usando Objetos Produto

Agora, em uma classe Loja, vamos criar instâncias de Produto e interagir com elas. 


public class Loja {
    public static void main(String[] args) {
        // Creating the first product object (instance of Produto).
        Produto produto1 = new Produto();
        produto1.nome = "Smartphone Modelo X";
        produto1.preco = 1999.90;
        produto1.quantidadeEmEstoque = 50;

        // Creating the second product object.
        Produto produto2 = new Produto();
        produto2.nome = "Notebook Gamer Y";
        produto2.preco = 4599.00;
        produto2.quantidadeEmEstoque = 25;

        // Calling the method on each object.
        System.out.println("Produtos disponíveis:");
        produto1.exibirFichaTecnica();
        produto2.exibirFichaTecnica();
    }
}

Exemplo 3: Método com Parâmetro e Retorno

Vamos adicionar um método à classe Produto que calcula o preço com desconto. Ele recebe o percentual de desconto como parâmetro e retorna o novo preço. 


// Add this method inside your Produto class.
/**
 * Calculates the product price with a given discount.
 * @param percentualDesconto The discount percentage (e.g., 10 for 10%).
 * @return The new price after applying the discount.
 */
public double calcularPrecoComDesconto(double percentualDesconto) {
    double valorDoDesconto = this.preco * (percentualDesconto / 100.0);
    return this.preco - valorDoDesconto;
}

Atividade Prática (TED): Modelando Alunos e Cursos

Vamos praticar a modelagem criando um sistema acadêmico simples. O objetivo é representar um Aluno e o Curso em que ele está matriculado.

  1. Crie a classe Curso.java com os atributos String nome e int duracaoHoras.
  2. Crie a classe Aluno.java com os atributos String nome, String matricula e Curso cursoMatriculado. Note que um atributo pode ser outro objeto!
  3. Adicione um método void exibirRelatorio() na classe Aluno que imprima o nome do aluno, sua matrícula e o nome do curso em que está matriculado.
  4. Crie uma classe Secretaria.java com um método main para testar. Nela, crie um objeto Curso (ex: "Análise e Desenvolvimento de Sistemas") e um objeto Aluno. Associe o curso ao aluno e chame o método exibirRelatorio().
