🚀 O Que é Programação Orientada a Objetos (POO)?

A Programação Orientada a Objetos (POO) é mais do que um conjunto de regras; é uma nova forma de pensar e estruturar soluções para problemas. Em vez de focarmos em procedimentos e lógica sequencial, a POO nos convida a modelar o mundo real através de "objetos".

Pense em um carro. Ele tem características (cor, marca, modelo) e pode realizar ações (ligar, acelerar, frear). Na POO, as características são os atributos e as ações são os métodos. Juntos, eles formam um objeto, uma entidade autônoma que encapsula seus próprios dados e comportamentos.

Paradigmas em Perspectiva

Programação Estruturada

Foco em funções e procedimentos. Os dados e as operações sobre os dados são separados.

  • Lógica: Top-down, sequencial.
  • Reuso: Limitado a funções.
  • Manutenção: Alterações podem ter impacto global.
  • Exemplo: Linguagem C.

Programação Orientada a Objetos

Foco em objetos que unem dados (atributos) e comportamentos (métodos).

  • Lógica: Modelagem do mundo real.
  • Reuso: Alto, através de herança e composição.
  • Manutenção: Modular, alterações são encapsuladas.
  • Exemplo: Java, C#, Python.

Boas Práticas para Iniciar em Java

Escrever código limpo desde o início acelera seu aprendizado e facilita a colaboração. Siga estas convenções:

  • Classes em PascalCase: CalculadoraDeImposto, Book.
  • Variáveis e Métodos em camelCase: pageCount, displayInfo().
  • Comentários Javadoc: Use /** ... */ para documentar classes e métodos, explicando o "porquê" do código.

🔑 Conceitos Fundamentais em Java

1. Classes e Objetos

A classe é o molde. O objeto é a instância concreta criada a partir do molde. Usamos a palavra-chave new para criar um objeto.


/**
 * Represents a book with a title, author, and page count.
 * This class serves as a blueprint for creating book objects.
 */
public class Book {
    String title;
    String author;
    int pageCount;

    /**
     * Displays the book's information to the console.
     * This is a 'void' method because it does not return any value.
     */
    public void displayInfo() {
        System.out.println("Title: " + title);
    }
}

2. Métodos com Retorno

Nem todos os métodos são void. Muitos realizam um cálculo ou uma operação e retornam um valor. Para isso, declaramos o tipo de retorno (ex: String, int) e usamos a palavra-chave return.


/**
 * A simple calculator to demonstrate methods that return values.
 */
public class Calculator {
    /**
     * Adds two integers and returns the result.
     * @param a The first number.
     * @param b The second number.
     * @return The sum of a and b.
     */
    public int add(int a, int b) {
        return a + b;
    }
}

// How to use it in the main method:
public class TestCalculator {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        int result = calc.add(5, 7); // The returned value (12) is stored in 'result'.
        System.out.println("The result is: " + result); // Prints "The result is: 12"
    }
}

Do Código-Fonte ao Programa: Compilação e Execução

Para que a JVM entenda nosso código Java, precisamos primeiro compilá-lo. Este processo transforma nosso arquivo .java em um arquivo .class (bytecode).

1. Compilar

javac NomeDaClasse.java

2. Executar

java NomeDaClasse

💡 Dica: Para listar arquivos no terminal, use ls (Linux/Mac) ou dir (Windows).

Atividade Prática (TED): Modelando seus Primeiros Objetos

Vamos aplicar os conceitos criando um pequeno cenário com duas classes que se relacionam: Motorista e Carro.

Parte 1: A Estrutura Básica

  1. Crie a classe Carro.java

    Um carro simples com marca, modelo e ano.

    
public class Carro {
    String marca;
    String modelo;
    int ano;
}
  2. Crie a classe Motorista.java

    O motorista terá um nome e um carro associado a ele.

    
public class Motorista {
    String nome;
    Carro carro; // A motorista "tem um" carro

    public void exibirInfo() {
        System.out.println("Motorista: " + nome);
        System.out.println("Carro: " + carro.marca + " " + carro.modelo);
    }
}
  3. Crie a classe Corrida.java para testar

    Aqui vamos criar os objetos e conectá-los.

    
public class Corrida {
    public static void main(String[] args) {
        // 1. Create a Carro object
        Carro carroDoJoao = new Carro();
        carroDoJoao.marca = "VW";
        carroDoJoao.modelo = "Nivus";
        carroDoJoao.ano = 2023;

        // 2. Create a Motorista object
        Motorista joao = new Motorista();
        joao.nome = "João";
        
        // 3. Associate the car with the driver
        joao.carro = carroDoJoao;

        // 4. Display the driver's info
        joao.exibirInfo();
    }
}

Parte 2 (Avançado): Construtores e Desafios

Agora, vamos refatorar o código usando construtores e adicionar um desafio.

  • Refatore as classes Carro e Motorista para usar construtores, como fizemos no exemplo da aula.
  • Desafio Opcional: Adicione um método acelerar() na classe Carro. Depois, crie uma sobrecarga: um acelerar() sem parâmetros e outro acelerar(int limiteDeVelocidade) que recebe um valor.
