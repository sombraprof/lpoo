Ementa e Contextualização

Nesta aula, faremos uma pausa estratégica para consolidar o conhecimento adquirido até agora, com a resolução comentada da AV1. Em seguida, daremos um passo fundamental na Programação Orientada a Objetos ao introduzir a Herança. A herança é um mecanismo poderoso que permite criar novas classes a partir de outras já existentes, promovendo o reuso de código, evitando duplicação e organizando nosso software em hierarquias lógicas. É o pilar que nos permite modelar relacionamentos do tipo "é-um" (um Gerente "é-um" Funcionário), essencial para a construção de sistemas complexos e escaláveis.

📚 Teoria — Análise da AV1: Consolidando Nossos Conhecimentos

Vamos revisar um problema típico da AV1, que aborda os conceitos essenciais que vimos até agora: classes, objetos, atributos, métodos e encapsulamento.

Problema Exemplo: Modelar um Curso

"Crie uma classe Curso para representar um curso acadêmico. A classe deve ter os atributos privados nome (String) e cargaHoraria (int). Implemente um construtor que inicialize esses atributos e métodos getters e setters. O setter de cargaHoraria deve garantir que o valor não seja negativo. Crie uma classe de teste para instanciar e exibir os dados de um curso."

Resolução Passo a Passo

A solução correta envolve criar a classe com atributos privados e fornecer acesso controlado através de métodos públicos.


// File: Curso.java
public class Curso {
    private String nome;
    private int cargaHoraria;

    public Curso(String nome, int cargaHoraria) {
        this.nome = nome;
        this.setCargaHoraria(cargaHoraria); // Use the setter to ensure validation
    }

    public String getNome() {
        return this.nome;
    }

    public int getCargaHoraria() {
        return this.cargaHoraria;
    }

    public void setNome(String nome) {
        this.nome = nome;
    }

    public void setCargaHoraria(int cargaHoraria) {
        if (cargaHoraria >= 0) {
            this.cargaHoraria = cargaHoraria;
        } else {
            System.out.println("[ERRO] Carga horária não pode ser negativa.");
        }
    }
}

🏗️ Construindo Hierarquias: O Poder da Herança

1. O que é Herança?

Herança é um mecanismo que permite que uma classe (subclasse) herde atributos e métodos de outra classe (superclasse). Isso cria uma relação "é-um". Por exemplo, um Cachorro é um Animal. A subclasse pode reutilizar todo o código da superclasse e ainda adicionar seus próprios atributos e métodos específicos.

  • Superclasse (ou Classe Pai/Base): A classe que está sendo herdada. Contém os membros mais genéricos.
  • Subclasse (ou Classe Filha/Derivada): A classe que herda. É uma versão especializada da superclasse.
  • Palavra-chave extends: Usada em Java para estabelecer a relação de herança. Ex: public class Cachorro extends Animal.

2. A Palavra-chave super

A palavra-chave super é usada dentro de uma subclasse para se referir a membros da sua superclasse. Ela tem dois usos principais:

  • super(): Para chamar o construtor da superclasse. Esta deve ser a primeira instrução no construtor da subclasse.
  • super.metodo(): Para chamar um método específico da superclasse, especialmente útil quando a subclasse sobrescreve esse método.

3. Vantagens e Cuidados com a Herança

Vantagens

  • Reuso de Código: Evita duplicação de atributos e métodos comuns.
  • Organização: Cria hierarquias lógicas e fáceis de entender.
  • Polimorfismo: Permite que objetos de subclasses sejam tratados como objetos da superclasse (veremos na próxima aula).

Cuidados

  • Acoplamento Forte: Subclasses são fortemente acopladas às suas superclasses. Mudanças na superclasse podem quebrar as subclasses.
  • Hierarquias Frágeis: Hierarquias muito profundas ou mal planejadas podem se tornar difíceis de manter.
  • Uso Indevido: Herança deve representar uma relação "é-um". Para relações "tem-um", prefira composição.

💻 Exemplos Práticos: Criando Hierarquias

Exemplo 1: Animal e Cachorro

Aqui, Cachorro herda nome de Animal e adiciona seu próprio atributo raca.


// Superclass
public class Animal {
    protected String nome;

    public Animal(String nome) {
        this.nome = nome;
    }

    public void comer() {
        System.out.println(this.nome + " está comendo.");
    }
}

// Subclass
public class Cachorro extends Animal {
    private String raca;

    public Cachorro(String nome, String raca) {
        // Calls the constructor of the superclass (Animal)
        super(nome); 
        this.raca = raca;
    }

    public void latir() {
        System.out.println(this.nome + " da raça " + this.raca + " está latindo: Au au!");
    }
}

Exemplo 2: Funcionario e Gerente

Um Gerente é um tipo de Funcionario que tem um bônus adicional.


// Superclass
public class Funcionario {
    private String nome;
    private double salario;

    public Funcionario(String nome, double salario) {
        this.nome = nome;
        this.salario = salario;
    }

    public double getSalario() {
        return this.salario;
    }
    
    public String getNome() {
        return this.nome;
    }
}

// Subclass
public class Gerente extends Funcionario {
    private double bonus;

    public Gerente(String nome, double salario, double bonus) {
        // Call the Funcionario constructor
        super(nome, salario);
        this.bonus = bonus;
    }

    // A better approach is to create a new method for the total salary.
    public double getSalarioComBonus() {
        // super.getSalario() calls the getSalario() method from the Funcionario class
        return super.getSalario() + this.bonus;
    }
}

Atividade Prática (TED): Modelando Veículos

Vamos praticar a herança modelando diferentes tipos de veículos.

  1. Crie a superclasse Veiculo.java com atributos private String marca e private String modelo. Adicione um construtor e getters para esses atributos.
  2. Crie a subclasse Carro.java que herda de Veiculo. Adicione um atributo private int numeroDePortas. Crie um construtor que use super() para inicializar a marca e o modelo.
  3. Crie a subclasse Moto.java que herda de Veiculo. Adicione um atributo private int cilindradas. Crie um construtor que também use super().
  4. Crie uma classe Garagem.java com um método main para testar. Crie um objeto Carro e um objeto Moto, e imprima as informações completas de cada um no console.
