Aprofundando em Herança e Polimorfismo

Aula 6: Unidade III - Conceitos práticos de OO e seus pilares

Ementa e Contextualização

Na aula anterior, introduzimos a Herança como um dos pilares da POO. Agora, vamos aprofundar nosso entendimento, explorando como as subclasses podem não apenas herdar, mas também modificar comportamentos através da sobrescrita de métodos. Isso nos levará diretamente ao conceito de Polimorfismo, talvez a característica mais poderosa da POO. Polimorfismo nos permite escrever um código mais flexível, genérico e extensível, tratando objetos de diferentes classes de maneira uniforme. Dominar esses conceitos é o que separa um programador iniciante de um arquiteto de software capaz de criar sistemas robustos e de fácil manutenção.

🚀 Elevando o Nível da Herança

1. Visibilidade Protegida (protected)

Até agora, usamos private para encapsulamento máximo. Mas e se quisermos que uma subclasse acesse diretamente um atributo da superclasse, sem que ele seja público para o resto do mundo? Para isso, usamos o modificador protected.

  • private: Acessível apenas dentro da própria classe.
  • protected: Acessível dentro da própria classe, por classes no mesmo pacote e por subclasses (mesmo em pacotes diferentes).
  • public: Acessível de qualquer lugar.

Nuance importante de protected

Fora do mesmo pacote, o acesso protected só é permitido em subclasses e, ainda assim, via a instância da subclasse (não via referência de Base). 


// English comments for clarity
// package base;
public class Base {
    protected int x = 42;
}

// package outro;
public class SubB extends Base {
    public void ok() {
        // OK: access via subclass instance (even if package is different)
        this.x = 10;
    }

    public void notOk() {
        Base b = new Base();
        // b.x; // ERROR: cannot access protected via superclass ref from another package
    }
}

2. Sobrescrita de Métodos (@Override)

Sobrescrever (ou anular) um método significa que uma subclasse fornece uma implementação específica para um método que já é fornecido por sua superclasse. A assinatura do método (nome e parâmetros) deve ser idêntica.

A anotação @Override

Embora opcional, é uma boa prática usar a anotação @Override acima de um método sobrescrito. Se houver erro na assinatura, o compilador alerta — evitando bugs sutis.

Overloading (compile-time)

  • Mesmo nome; assinaturas diferentes.
  • Resolução em tempo de compilação.
  • Não envolve herança, em geral.

Overriding (runtime)

  • Mesma assinatura em subclasse.
  • Binding dinâmico em execução.
  • Permite polimorfismo real.

3. Polimorfismo: Múltiplas Formas

Polimorfismo significa "muitas formas". Em POO, é a capacidade de um objeto ser referenciado de múltiplas maneiras. A forma mais comum é usar uma variável de referência do tipo da superclasse para apontar para um objeto da subclasse. 


// English comment: Upcasting allows superclass reference to point to subclass object
Funcionario f = new Gerente("Ana", 8000.0, 1500.0);

A "mágica" acontece quando chamamos um método sobrescrito a partir dessa referência. O Java, em tempo de execução, sabe qual é o tipo real do objeto (Gerente) e chama a versão correta.

Downcasting seguro com instanceof (pattern matching)


// English comments: Pattern matching for instanceof (Java 16+)
Funcionario ref = new Gerente("Maria", 5000.0);
if (ref instanceof Gerente gerente) { // auto-cast bound to variable
    System.out.println("Bônus especial: " + gerente.getBonificacao());
}

4. A Classe Object: A Mãe de Todas

Em Java, toda classe herda implicitamente de Object. Isso garante métodos como equals(), hashCode() e toString().

É uma ótima prática sobrescrever toString() para representações textuais úteis.

Contrato equals() / hashCode()

Ao sobrescrever equals(), também sobrescreva hashCode() de forma consistente (coerência em coleções como HashSet/HashMap).

Prefira composição a herança

Use herança quando respeitar o princípio de substituição de Liskov (LSP) e houver uma relação "é-um". Em muitos casos, composição oferece acoplamento menor e maior reuso.

💻 Praticando Polimorfismo

Vamos evoluir nosso exemplo Funcionario/Gerente. Adicionaremos um Programador e usaremos polimorfismo para calcular uma folha de pagamento. 


// Superclass Funcionario (educational example)
public class Funcionario {
    protected String nome;   // For teaching purposes; in production prefer private + getters
    protected double salario;

    public Funcionario(String nome, double salario) {
        this.nome = nome;
        this.salario = salario;
    }

    // A general bonus calculation
    public double getBonificacao() {
        return this.salario * 0.05; // 5% bonus for a regular employee
    }
    
    @Override
    public String toString() {
        return "Funcionario [Nome=" + this.nome + ", Salário=" + this.salario + "]";
    }
}

// Subclass Gerente
public class Gerente extends Funcionario {
    public Gerente(String nome, double salario) {
        super(nome, salario);
    }

    @Override // Overriding the method for a specific implementation
    public double getBonificacao() {
        return super.getBonificacao() + 1000.0; // 5% + a fixed bonus of 1000
    }
}

// Subclass Programador
public class Programador extends Funcionario {
     public Programador(String nome, double salario) {
        super(nome, salario);
    }
    
    @Override // Specific bonus for programmers
    public double getBonificacao() {
        return this.salario * 0.15; // 15% bonus for programmers
    }
}

// Test Class
import java.util.List;

public class ControleBonificacao {
    public static void main(String[] args) {
        // Polymorphism in action: Superclass reference for subclass objects
        Funcionario f1 = new Funcionario("João", 2000.0);
        Funcionario g1 = new Gerente("Maria", 5000.0);
        Funcionario p1 = new Programador("Pedro", 3500.0);

        System.out.println("Bônus do Funcionário: " + f1.getBonificacao()); // Calls Funcionario's method
        System.out.println("Bônus do Gerente: " + g1.getBonificacao());     // Calls Gerente's method
        System.out.println("Bônus do Programador: " + p1.getBonificacao()); // Calls Programador's method
        
        System.out.println(f1); // Prints the result of the overridden toString() method
        System.out.println(g1);

        // Optional: polymorphic collections
        List funcionarios = List.of(f1, g1, p1);
        double total = 0;
        for (Funcionario f : funcionarios) {
            total += f.getBonificacao(); // runtime dispatch
        }
        System.out.println("Total de bonificações: " + total);
    }
}

Nota: para valores monetários em produção, prefira BigDecimal.

Atividade Prática (TED): Figuras Geométricas

Vamos aplicar a herança e o polimorfismo para calcular a área de diferentes formas geométricas.

  1. Crie a superclasse abstrata FormaGeometrica.java. Ela deve declarar o método public abstract double calcularArea().
  2. Crie a subclasse Retangulo.java que herda de FormaGeometrica. Adicione os atributos largura e altura. Implemente calcularArea() retornando largura * altura.
  3. Crie a subclasse Circulo.java que herda de FormaGeometrica. Adicione o atributo raio. Implemente calcularArea() retornando Math.PI * raio * raio.
  4. Crie a classe TesteFormas.java com um método main. Crie uma coleção (List<FormaGeometrica>) com instâncias de Retangulo e Circulo. Percorra e imprima a área de cada forma, observando o polimorfismo em ação (binding dinâmico). 

// English comments: Abstract base class to enforce area calculation
public abstract class FormaGeometrica {
    public abstract double calcularArea();

    @Override
    public String toString() {
        return getClass().getSimpleName();
    }
}

public class Retangulo extends FormaGeometrica {
    private final double largura;
    private final double altura;

    public Retangulo(double largura, double altura) {
        this.largura = largura;
        this.altura = altura;
    }

    @Override
    public double calcularArea() {
        return largura * altura;
    }
}

public class Circulo extends FormaGeometrica {
    private final double raio;

    public Circulo(double raio) {
        this.raio = raio;
    }

    @Override
    public double calcularArea() {
        return Math.PI * raio * raio;
    }
}

import java.util.List;

public class TesteFormas {
    public static void main(String[] args) {
        List formas = List.of(
            new Retangulo(3, 4),
            new Circulo(2.5)
        );

        for (FormaGeometrica f : formas) {
            System.out.printf("%s -> área = %.2f%n", f, f.calcularArea());
        }
    }
}
