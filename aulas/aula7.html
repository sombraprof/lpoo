Classes Abstratas e Interfaces

Aula 7: Unidade III - Conceitos práticos de OO e seus pilares

Ementa e Contextualização

Até agora, trabalhamos com classes concretas, que podem ser instanciadas diretamente. No entanto, ao modelar sistemas complexos, muitas vezes precisamos de um nível maior de abstração. As classes abstratas nos permitem criar "modelos" de classes incompletas, forçando as subclasses a fornecerem implementações específicas. Já as interfaces levam a abstração a um novo patamar, definindo "contratos" de comportamento que classes completamente diferentes podem seguir. Dominar esses dois conceitos é crucial para projetar sistemas desacoplados, flexíveis e que seguem os princípios de design SOLID.

Exemplo 3: Conflito de default em Múltiplas Interfaces

Quando duas interfaces definem o mesmo método default, a classe precisa resolver o conflito explicitamente.

interface A { default void log() { System.out.println("A"); } }
interface B { default void log() { System.out.println("B"); } }

class C implements A, B {
    @Override // Must resolve the conflict explicitly
    public void log() {
        A.super.log(); // or B.super.log(); or custom logic
        System.out.println("C");
    }
}

Extras Modernos: Hierarquias sealed (Java 17+)

Interfaces e classes podem ser seladas para restringir quem pode implementá-las/herdá-las.

public sealed interface Voavel permits Passaro, Aviao {
    void voar();
}

public final class Passaro implements Voavel { // final: cannot be subclassed further
    @Override public void voar() { System.out.println("Batendo asas"); }
}

public non-sealed class Aviao implements Voavel {
    @Override public void voar() { System.out.println("Turbinas ligadas"); }
}

📜 Definindo Contratos e Modelos

1. Classes Abstratas: O Modelo Incompleto

Uma classe abstrata é uma classe que não pode ser instanciada. Ela serve como um modelo base para outras classes. Pode conter tanto métodos concretos (com implementação) quanto métodos abstratos (sem implementação).

  • Palavra-chave abstract: Usada para declarar tanto a classe quanto os métodos abstratos.
  • Métodos Abstratos: São como uma promessa. Eles declaram uma assinatura de método, mas sem corpo. Qualquer classe concreta que herda da classe abstrata é obrigada a implementar todos os métodos abstratos.
  • Quando usar? Quando você tem uma relação "é-um" (herança) e quer fornecer um comportamento base comum para todas as subclasses, mas ao mesmo tempo forçar que cada uma delas implemente uma parte específica do comportamento.

2. Interfaces: O Contrato de Comportamento

Uma interface é uma referência de tipo completamente abstrata. Ela só pode conter assinaturas de métodos e constantes (public static final). Uma classe pode implementar uma ou mais interfaces.

  • Palavra-chave interface e implements: interface para declarar e implements para uma classe implementar a interface.
  • 100% Abstrata (antes do Java 8): Todos os métodos são implicitamente public e abstract. Todos os atributos são public static final.
  • Quando usar? Quando você quer definir um "contrato" de comportamento que pode ser implementado por classes de diferentes hierarquias. É a base para um design de software desacoplado e a forma como Java simula a "herança múltipla".

Classe Abstrata vs. Interface

Característica Classe Abstrata Interface (Java 8+)
Instanciação Não pode ser instanciada; pode ter construtor (chamado via super()). Não pode ser instanciada; sem construtor.
Métodos Pode ter métodos abstratos e concretos. Abstratos, default, static e (Java 9+) private.
Atributos/Estado Pode ter qualquer tipo de atributo (static, final, non-final). Só pode ter constantes (public static final).
Herança Uma classe pode herdar de apenas uma classe abstrata (extends). Pode extends múltiplas interfaces (herança múltipla de tipo).
Construtor Pode ter construtores (chamados via super() pelas subclasses). Não pode ter construtores.
Propósito Definir um modelo base com comportamento comum para uma hierarquia (relação "é-um"). Definir um contrato de capacidades que podem ser compartilhadas por classes não relacionadas.
Extras modernos Pode ser sealed (Java 17+) para controlar subclasses. Conflitos de default; @FunctionalInterface; também pode ser sealed.

💻 Abstração na Prática

Exemplo 1: Classe Abstrata FormaGeometrica

FormaGeometrica define o conceito de "área", mas não sabe como calculá-la. Retangulo e Circulo são obrigados a fornecer a fórmula. 

// Abstract base class with state and constructor
public abstract class FormaGeometrica {
    private final String nome;

    public FormaGeometrica(String nome) {
        this.nome = nome;
    }

    public String getNome() { // Exposes read-only state
        return nome;
    }

    public abstract double calcularArea(); // Must be implemented by subclasses
}

public class Retangulo extends FormaGeometrica {
    private final double largura;
    private final double altura;

    public Retangulo(double largura, double altura) {
        super("Retangulo");
        this.largura = largura;
        this.altura = altura;
    }

    @Override
    public double calcularArea() {
        return largura * altura;
    }
}

public class Circulo extends FormaGeometrica {
    private final double raio;

    public Circulo(double raio) {
        super("Circulo");
        this.raio = raio;
    }

    @Override
    public double calcularArea() {
        return Math.PI * raio * raio;
    }
}

Exemplo 2: Interface Tributavel

Tanto uma ContaCorrente quanto um SeguroDeVida podem ser tributáveis, mas não têm uma relação de herança. A interface Tributavel cria um contrato comum. 

// Contract for taxable entities (Java 8+ features demonstrated)
public interface Tributavel {
    double getValorImposto();             // Abstract method (must be implemented)

    default double aplicarAcrescimo(double valor) {
        // Default method provides reusable behavior for implementors
        return valor + calcularAcrescimo(valor);
    }

    private double calcularAcrescimo(double valor) { // Java 9+ private reuse
        return valor * 0.02; // 2% extra fee as an example
    }

    static double arredonda(double v) {   // Utility static method
        return Math.round(v * 100.0) / 100.0;
    }
}

// Simple account (educational)
class Conta {
    private double saldo;
    public Conta(double saldo) { this.saldo = saldo; }
    public double getSaldo() { return saldo; }
}

class ContaCorrente extends Conta implements Tributavel {
    public ContaCorrente(double saldo) { super(saldo); }
    @Override public double getValorImposto() {
        return getSaldo() * 0.01; // 1% of balance
    }
}

class SeguroDeVida implements Tributavel {
    @Override public double getValorImposto() {
        return 42.0; // fixed tax
    }
}

import java.util.List;

class CalculadoraImpostos {
    public static void main(String[] args) {
        // Polymorphism with interfaces: unrelated classes share a contract
        List itens = List.of(
            new ContaCorrente(2500.0),
            new SeguroDeVida()
        );

        double total = 0;
        for (Tributavel t : itens) {
            double base = t.getValorImposto();
            // Using default + static methods from the interface
            double comAcrescimo = Tributavel.arredonda(t.aplicarAcrescimo(base));
            total += comAcrescimo;
            System.out.println("Item -> imposto: " + comAcrescimo);
        }
        System.out.println("Total: " + Tributavel.arredonda(total));
    }
}

Atividade Prática (TED): Modelando um Zoológico

Vamos projetar um sistema simples para um zoológico, combinando classes abstratas e interfaces.

  1. Crie a classe abstrata Animal.java. Ela deve ter um atributo nome e um método abstrato public abstract void emitirSom().
  2. Crie as classes concretas Leao.java e Passaro.java, ambas herdando de Animal. Implemente o método emitirSom() em cada uma (ex: "Rugido!" e "Pio!").
  3. Crie a interface Voavel.java. Ela deve ter um único método: public void voar().
  4. Faça a classe Passaro implementar a interface Voavel e forneça uma implementação para o método voar(). O Leão não deve implementar esta interface.
  5. Crie uma classe Zoologico.java com um método main. Crie um Leao e um Passaro. Use polimorfismo para chamar emitirSom() de ambos. Verifique se o pássaro pode voar, mas o leão não.
